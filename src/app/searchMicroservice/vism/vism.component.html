5.3.2.1. Modelo De Entidad Relación

Este es el modelo más sencillo. Al ser una base de datos no relacional, sólo se va a tener un tipo de documento o colección de documentos, el ParquingLot. Éste tiene la información necesaria para poder consultar los parqueaderos disponibles con base en los criterios descritos en la ASR 11. Los siguientes atributos van a ir indexados para agilizar las búsquedas: zone, vehicleType y location.
