1. Introducción

Esta sección pretende dar una visión general de todo Documento de Arquitectura de Software, llamado de aquí en adelante SAD. En esta sección se explica el propósito, alcance, definiciones, referencias y visión general del documento.
1.1. Objetivo

Este documento es actualizable de acuerdo a los cambios técnicos de arquitectura que vayan apareciendo. Su objetivo es mostrar de forma organizada la arquitectura de software definida para el desarrollo del actualización del sistema NIDOO, que es el producto del trabajo realizado por los autores del documento para el proyecto del curso arquitecturas de software ágiles. Con este documento, se pretende plasmar en términos arquitectónicos y de diseño todos los requerimientos definidos en el documento del proyecto del curso, entregado por el cliente.

Este documento es la principal fuente de información para empezar con el desarrollo del sistema NIDOO, así como la integración de cada uno de los componentes presentados en el documento y va dirigido a los stakeholders que harán parte del proceso de desarrollo del software. (CTO, Arquitectos de software, gerente de proyecto, ingenieros de requerimientos, líder técnico, desarrolladores y tester).

1.2. Alcance

Este proyecto se desarrolla como requisito de la materia de software desarrollos ágiles dirigida por el profesor Dario Ernesto Correal Torres, Phd en Ingeniería de la Universidad De Los Andes. Con este documento se pretende definir y proponer una arquitectura del software NIDOO donde posteriormente la representación arquitectónica estará especificada por sus vistas, módulos, técnicas y tácticas definidas para el sistema acorde a los requerimientos establecidos en el documento del proyecto.

También son detallados y especificados otros aspectos importantes del proceso como son las historias de usuario, ASR, sprints, experimentos de algunos atributos de calidad y sus conclusiones.

En este documento se presentan diferentes módulos a desarrollar y la manera en que éstos se relacionan con los componentes propuestos. El sistema contará con las funcionalidades definidas y mapeadas al software en este documento.
