5.6. Vista De Operación

5.6.1. Modelo De Asignación

Los ambientes operacionales del sistema Nidoo se asignarán cada uno como un proyecto independiente en Heroku. Estos ambientes serán utilizados durante el proceso de desarrollo de los microservicios y cada uno tendrá una línea de desarrollo (branch) para ser desplegados de forma controlada a través del repositorio repositorio de versionamiento (Git).

Procedimiento de asignación:
  • Para cada ambiente se debe crear un nuevo proyecto en Heroku. Este proceso debe ser realizado por el administrador del PaaS en Nidoo.
  • Para cada ambiente se deben asignar los gadgets requeridos para el proyecto: RabbitMQ, New relic, MongoDB, Logentries (Se debe tener en cuenta el costo asociado a estos gadgets).
Procedimiento de usuarios:
  • Se deben registrar en Heroku los usuarios segun su rol (Desarrollador, tester, arquitecto, gerente de proyecto, etc).
  • Asignar permisos y privilegios de acceso a los usuarios para los componentes definidos por el arquitecto.
