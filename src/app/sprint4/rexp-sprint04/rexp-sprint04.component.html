Resultados Del Experimento Sprint 04

En esta sección presentamos los resultados obtenidos del experimento de los puntos sensibilidad que definimos para el sprint 04 y que están enfocados en el atributo de calidad de Seguridad.

Análisis De Los Resultados Obtenidos

Para el experimento se montó y configuró un API Gateway (Kong), se creó un nuevo microservicio payments con un nuevo servicio POST /zonefee, que se encarga de agregar una nueva tarifa para una zona en particular, y se subió a Heroku para hacer las pruebas. También se aprovisionó de un proveedor de Mail llamado Sendgrid, MongoLab para la base de datos y LogEntries para manejo de logs. Todo esto con el fin de probar que el cambio de estas tarifas, al ser una operación sensible, sólo personas autorizadas y por medio del servicio expuestos pueden hacerlo. Igualmente de haberlo cambiado por una persona con acceso, quedaría registro en los logs para auditoria y darle trazabilidad al cambio. A parte debe quedar un registro adicional en la base de datos y si por algún motivo quien tenga acceso directo a la base de datos hace el cambio sobre la colección, la base de datos dispara un trigger con un correo notificando de los cambios realizados. A parte el servicio envía una notificación para que haya redundancia por si alguno de los mecanismos instaurados llega a ser vulnerado.
