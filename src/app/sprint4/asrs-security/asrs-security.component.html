ASRs Consideradas En El Sprint 4

En esta sección presentamos los ASR consideradas para satisfacer los requerimientos de seguridad de la arquitectura. El objetivo es determinar los puntos de sensibilidad para determinar si las decisiones de arquitectura que se han tomado van por buen camino.
ASRs Consideradas

Código Descripción
ASR08 Como oferente, cuando registre un parqueadero, dado que el sistema opera normalmente, quiero que se cobre la tarifa estipulada para la zona en la que se encuentra. Ésta no debe ser cambiada por alguien no autorizado y siempre se debe cobrar el valor configurado.
ASR13 Como usuario cuando esté ingresando medios de pago en el sistema, dado que este funciona normalmente, quiero que me aseguren que mi información estará protegida y nadie más podrá acceder a ella el 100% de las veces.
ASR23 Como oferente, cuando quiera pagar mi parqueadero dado que el sistema opera normalmente, quiero que la información de mis métodos de pago esté asegurada para que nadie pueda accederla.
ASR24 Como administrador del sistema, cuando alguien quiera acceder a la base de datos de sistema dado que opera normalmente, quiero que contemos con las políticas de seguridad de accesos, para evitar fugas de información.
