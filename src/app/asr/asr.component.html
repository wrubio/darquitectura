4.3. Requisitos de Importancia Arquitectónica ASR

Esta sección se presentan aquellos requisitos que tienen un gran impacto o efecto importante en la arquitectura del sistema, de manera que si éstas están ausentes, la arquitectura resultante sería diferente.

Para la definición de cada una de estas, se realizó una categorización de las decisiones de diseño de la arquitectura, donde se buscó cuál de los requisitos del proyecto podría afectar directamente las decisiones de arquitectura.
4.3.1. Tabla De ASRs

Código Descripción
ASR01 Como propietario, cuando me lleguen notificaciones dado que el sistema opera normalmente quiero ver y modificar el estado del parqueadero para validar y actualizar la disponibilidad. Esto debe suceder con 100.000 usuario concurrentes en menos de 30 segundos.​
ASR02 Como propietario, cuando cargue los datos de parqueaderos dado que el sistema opera en modo normal quiero que aparezca la oferta para que los clientes puedan visualizarlas disponibles. Esto debe suceder en el 99,96 % de las veces.​
ASR03 Cuando se requieran autorizaciones de ingreso, dado que el sistema opera normalmente quiero poder subir las autorizaciones al sistema para que estén disponibles para el cliente y agilizar el proceso de ingreso, esto debe suceder el 99.99% de las veces.​
ASR04 Como oferente, cuando consulte la información de mi estado de dado que el sistema opera normalmente, quiero la información no muestre medios de pago ni información de cuentas.​
ASR05 Como oferente corporativo, quiero poder validar la disponibilidad de mis parqueaderos en el dashboard y tener una respuesta en menos de 2 segundos.
ASR06 Como oferente quero que la plataforma me envíe una notificación cuando alguien desea alquilar mi parqueadero en menos de 5 segundos después de que el sistema haya recibido la solicitud​.
ASR07 Cuando se hagan solicitudes de reserva del parqueadero, dado que el sistema opera normalmente quiero ver la información del cliente NIDOO para agilizar el proceso ingreso, esto debe suceder el 99.99% de las veces.​
ASR08 Como oferente corporativo cuando revise las tarifas, dado que el sistema opera normalmente, quiero que estas sean proporcionales según su ubicación y que los demás parqueaderos no puedan modificar sus tarifas para evitar problemas legales y de competencia. Esto debe suceder el 100% de las veces.
ASR09 Como usuario, cuando realice una búsqueda dado que el sistema opera normalmente quiero buscar parqueaderos dados los criterios: (Dirección, zona, tipo vehículo) para ver los parqueaderos más cercanos. Esto debe suceder con 100.000 usuario concurrentes en menos de 5 segundos.
ASR10 Como usuario, cuando aplique a una reserva dado que el sistema opera en modo sobrecarga quiero poder reservar un parqueadero para dejar mi carro cerca al trabajo. Esto debe suceder entre 3 a 5 segundos.
ASR11 Como usuario, cuando necesite cancelar mi reserva, dado que el sistema opera normalmente, quiero que el sistema me responda entre 2 y 3 segundos si la cancelación fue exitosa o no.
ASR12 Cuando cree mi cuenta de NIDOO dado que el sistema opera normalmente, quiero que el sistema me permita usar la misma cuenta para cuando salga del país. Esto debe suceder el 99.5% de las veces.
ASR13 Como usuario cuando esté ingresando medios de pago en el sistema, dado que este funciona normalmente, quiero que me aseguren que mi información estará protegida y nadie más podrá acceder a ella el 100% de las veces.
ASR14 Como administrador de parqueadero, dado que el sistema opera normalmente, quiero recibir una notificación de que el cliente ya está llegando para poder abrirle la puerta en menos de 5 segundos.
ASR15 Como administrador de parqueadero, dado que el sistema opera normalmente, quiero poder validar la identidad del conductor a ingresar y que el sistema me responsa en menos de 2 segundos.
ASR16 Como usuario, cuando realice una búsqueda dado que el sistema opera normalmente quiero buscar parqueaderos dados los criterios: (Dirección, zona, tipo vehículo) para ver los parqueaderos más cercanos. Esto debe suceder entre 1 y 2 segundos.
ASR17 Como usuario, cuando aplique a una reserva dado que el sistema opera en modo sobrecarga quiero poder reservar un parqueadero para dejar mi carro cerca al trabajo. Esto debe suceder con 2.000 usuarios concurrentes por segundo durante 5 segundos.
ASR18 Como usuario, cuando necesite cancelar mi reserva, dado que el sistema opera normalmente, quiero que el sistema me responda en menos de 3 segundos si la cancelación fue exitosa o no. Esto debe suceder en con 1000 usuarios concurrentes por segundo durante 5 segundos.
ASR19 Como usuario, cuando realice una búsqueda dado que el sistema opera normalmente quiero buscar parqueaderos dados los criterios: (Dirección, zona, tipo vehículo) para ver los parqueaderos más cercanos. Esto debe suceder en menos de 1 segundo con 2000 usuarios concurrentes durante 5 segundos.
